JDBC（Java连接数据库）学习笔记

JDBC+DAO（Java连接数据库）学习笔记

一：JDBC基本接口类介绍+如何使用

01：DriverManager类介绍

  • 该类负责管理各种不同的JDBC驱动
  • 连接不同的数据库时需要用DriverManager管理加载不同的驱动地址（jar包地址.Driver）

02：Connection接口介绍

  • 该接口负责连接数据库并担任传输数据的任务
  • 使用该接口获取数据库地址，连接用户名，连接密码之后连接到目标数据库

03：Statement接口介绍

  • 该接口由Connection产生，负责执行SQL语句
  • 使用该接口对象调用execute[….]（SQL语句）等一系列方法执行SQL语句获取想得到的效果

04：ResultSet接口介绍

  • 该接口负责接收处理Statement执行SQL后返回的结果
  • 可以用该接口的对象调用next，get…(标识)等一系列方法获取其当中的值

05：PreparedStatement接口介绍

  • 该接口继承至Statement，同样是负责SQL语句的执行
  • 该接口可以使用conn.prepareStatement(sql)预先编译SQL再填充部分占位值，然后使用execute[….]（SQL语句）等一系列方法执行SQL语句获取想得到的效果，比起Statement接口也可以更灵活安全的执行SQL
  • 程序执行SQL推荐用该接口执行

07：查询操作详解(在操作前记得要配置相关驱动jar包)

08：SQL语句字符串注意事项

二：DAO模式详解

01：数据持久化

  • 将程序中的数据在瞬时状态和持久状态间转换的机制即为数据持久化
  • 数据的持久化方式一般有：数据库存储，文件存储和XML文件存储等等……

02:DAO概要

  • 使用面向接口编程的模式将JDBC封装的结构被称之为DAO模式
  • DAO模式可以降低代码间的耦合性，增强代码的复用性与可维护性，同时也能进一步的保证程序的安全
  • DAO模式组成：DAO接口 + DAO实现类 + 实体类 + 数据库连接和关闭工具类（可在其中定义公共增删改方法）
  • DAO主要就是定义所需要的SQL操作服务接口，再由实现类去实现其中的服务方法，DAO公共类是为了增强代码复用存在，将增删改，获取连接和关闭资源定义为方法，以便实现代码复用

三：Properties类概要使用详解

01：Properties概要

  • 让用户脱离程序本身修改相关的变量设置——使用配置文件（properties文件）
  • 在配置文件中设置jdbc所需要的各变量值，就可以无需重新编译就可以修改数据
  • 在Java中通过Properties类中的方法配合IO流就可以获取到配置文件中的数据信息

02：实际操作

  • ?useUnicode=true&characterEncoding=当前程序字符集可以防止编程软件在和数据库进行数据交接时会出现的编码问题
